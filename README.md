# WEM Interactive Sales Deck

A cinematic, interactive digital experience showcasing commercial opportunities at West Edmonton Mall. Designed with a luxury-grade aesthetic and optimized for high-performance engagement.

## 🚀 Live Demo
[View Live Site](https://shauryasingh2093.github.io/mall_salesdeck/) (Ensure GitHub Pages is enabled in repo settings)

## ✨ Key Features
- **Interactive Navigation**: Non-linear, viewer-controlled journey through 8 distinct layers of the WEM ecosystem.
- **Video-First Storytelling**: Cinematic background videos and optimized visual assets.
- **Luxury UI**: Minimalist, high-impact design inspired by global brands like Apple and Tesla.
- **Fast & Performant**: Implements lazy loading, video preloading, and optimized asset delivery for a smooth user experience.
- **Responsive Design**: Flawless performance across Desktop and Tablet viewports.
- **Interactive Components**: Custom-built experience carousel and image lightbox system.

## 🛠️ Tech Stack
- **Frontend**: Vanilla HTML5, CSS3 (Modern Flexbox/Grid), JavaScript (ES6+).
- **Assets**: High-resolution imagery and AI-generated renderings for a premium feel.
- **Fonts**: Inter & Cormorant Garamond (via Google Fonts).

## 📦 Getting Started

### Local Development
1. Clone the repository:
   ```bash
   git clone https://github.com/shauryasingh2093/mall_salesdeck.git
   ```
2. Open `index.html` in your browser.
   - *Note: For the best experience, use a local server (like Live Server in VS Code) to ensure all assets load correctly.*

### Deployment Instructions

#### GitHub Pages (Recommended)
1. Push your code to the `main` branch on GitHub.
2. Go to your repository on GitHub.com.
3. Click on **Settings** > **Pages**.
4. Under **Build and deployment** > **Branch**, select `main` and `/ (root)`.
5. Click **Save**. Your site will be live at `https://<username>.github.io/mall_salesdeck/` within minutes.

#### Vercel / Netlify
1. Connect your GitHub repository to [Vercel](https://vercel.com) or [Netlify](https://netlify.com).
2. The platform will automatically detect the static site and deploy it.
3. No build commands are required as this is a vanilla HTML/CSS/JS project.
